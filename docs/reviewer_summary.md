## Why It Matters

The project targets a practical Godot ecosystem gap: many examples show isolated mechanics, but fewer provide a runnable, inspectable mobile-first loop that combines touch input, portrait readability, auto-combat, pickups, upgrade choices, relic-style modifiers, encounter pacing, and iOS export considerations.

## Maintenance Evidence

## Current Limitations

The project is early-stage and does not claim broad adoption yet. The application positioning is based on ecosystem usefulness, active maintenance, and clear reference value for Godot mobile action-game developers.
